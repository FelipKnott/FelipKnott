### `> cat methodology.md`

```
Workflow: Senior Developer — 10-Phase Engineering Discipline

Phase 0 → Mindset: "Plan more than you code"
Phase 1 → Discovery: Investigate BEFORE any code
Phase 2 → Documentation: ADRs + 70+ MDs per complex project
Phase 3 → Planning: Fibonacci story points, 35+ acceptance criteria
Phase 4 → Implementation: TDD Red-Green-Refactor (minutes, not hours)
Phase 5 → Quality Gates: 90%+ coverage PER FILE
Phase 6 → Stress Test: p95 < 500ms, error rate < 1%
Phase 7 → QA: Gherkin BDD, all paths tested (not just happy path)
Phase 8 → Deploy: Atomic commits, feature branches, never push to main
Phase 9 → Post-Production: Immutable source of truth, regression tests

Rule: No code is written before
  Discovery → Documentation → ADR → Planning → Acceptance Criteria

"It's easy to use → It's hard to build.
 It's hard to use → It's easy to build."
```
